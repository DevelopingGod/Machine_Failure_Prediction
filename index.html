About Dataset¶

Dataset Overview¶

This dataset contains sensor data collected from various machines, with the aim of predicting machine failures in advance. It includes a variety of sensor readings as well as the recorded machine failures.

Columns Description¶

  • footfall: The number of people or objects passing by the machine.

  • temp Mode: The temperature mode or setting of the machine.

  • AQ: Air quality index near the machine.

  • USS: Ultrasonic sensor data, indicating proximity measurements.

  • CS: Current sensor readings, indicating the electrical current usage of the machine.

  • VOC: Volatile organic compounds level detected near the machine.

  • RP: Rotational position or RPM (revolutions per minute) of the machine parts.

  • IP: Input pressure to the machine.

  • Temperature: The operating temperature of the machine.

  • fail: Binary indicator of machine failure (1 for failure, 0 for no failure).

In summary, main task is to predict whether a machine will fail (fail column) using the sensor data as features.¶

Notes: [Very important, please read]¶

1] I have used the following ways¶

  • Logistic Regression
  • Logistic Regression with Hyper-Parameter Tuning

2] Which parameter is more important? It's recall, here's why:-¶

  • Recall focuses on the proportion of actual failures (positive cases) that model correctly identifies.

  • In a machine failure prediction scenario, missing a failure (False Negative) could be costly or dangerous, as it might lead to an undetected machine failure

  • EG. If a machine is predicted to pass (0) but actually fails (1), this is a False Negative, and such an error could lead to serious consequences.

  • By optimizing for recall, we ensure that model catches as many failures as possible, even if that means sometimes predicting a failure when there isnt one (False Positive).

  • This could lead to more preventive maintenance, which might be preferable over the risk of missing a failure.
